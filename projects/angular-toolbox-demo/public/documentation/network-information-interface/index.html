NetworkInformation Interface

Since v 0.10.0

The NetworkInformation interface of the Network Information API provides information about the connection a device is using to communicate with the network and provides a means for scripts to be notified if the connection type changes. The NetworkInformation interface cannot be instantiated. It is instead accessed through the connection property of the Navigator interface or the WorkerNavigator interface.

NetworkInformation.downlink

readonly downlink: number

Returns the effective bandwidth estimate in megabits per second, rounded to the nearest multiple of 25 kilobits per seconds.

NetworkInformation.effectiveType

readonly effectiveType: EffectiveConnectionType

Returns the effective type of the connection meaning one of the EffectiveConnectionType enum. This value is determined using a combination of recently observed round-trip time and downlink values.

NetworkInformation.type

readonly type: ConnectionType

Returns the type of connection a device is using to communicate with the network. It will be one of the ConnectionType enum.

NetworkInformation.rtt

readonly rtt: number

Returns the estimated effective round-trip time of the current connection, rounded to the nearest multiple of 25 milliseconds.

NetworkInformation.saveData

readonly saveData: boolean

Returns true whether the user has set a reduced data usage option on the user agent. false otherwise.

NetworkInformation.onchange

onchange: EventListener

The event that is fired when connection information changes.

EffectiveConnectionType

The effective type of connection as defined by the NetworkInformation.effectiveType property.
Possible values are:

  • 2g
  • 3g
  • 4g
  • slow-2g

ConnectionType

The type of connection as defined by the NetworkInformation.type property.
Possible values are:

  • bluetooth
  • cellular
  • ethernet
  • mixed
  • none
  • other
  • unknown
  • wifi
  • wimax
