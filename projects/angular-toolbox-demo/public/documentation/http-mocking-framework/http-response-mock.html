HTTP Mocking Framework

The HTTP Mocking Framework provides an easy-to-use and highly configurable solution to mock HTTP responses to API calls sent from your Angular applications.

HttpResponseMock Interface

The HttpResponseMock interface defines the config of a HTTP response mock object. The mock response objects contain the return values expected by the client after initiating an HTTP request.

HttpResponseMock.body

body?: any | Observable<any>

The body content of the HTTP response. Can be either an object, or an Observable instance.
Default value is null.

HttpResponseMock.headers

headers?: HttpHeaders

An optional property that defines all headers associated with the HTTP response.
See the HttpHeaders reference for more details.

HttpResponseMock.status

status: number

The status code of the HTTP response.
See the HttpStatusCode reference for more details.

HttpResponseMock.statusText

statusText: string

The textual description of the HTTP response status code.

HttpResponseMock.url

url: string | null

The URL of the resource retrieved, or null if not available.

HttpResponseMock.error

error: HttpMockError | null

An optional error for this HTTP request.
Default value is null.
See the HttpMockError interface reference for details.

HttpResponseMock.delay

delay?: number

Defines the delay of the HTTP response, in miliseconds.
Maximum value is 10000 miliseconds (10 seconds).
Default value is 0.
