VERSION_CONFIG Provider

The default provider for the VersionService configuration. You typically define the custom properties in the main NgModule declaration to initialize the app version.

VersionConfig.major

major = 0

Indicates the app version when you make incompatible API changes.
Default value is 0.

VersionConfig.minor

minor = 0

Indicates the app version when you add functionality in a backward compatible manner.
Default value is 0.

VersionConfig.patch

patch = 0

Indicates the app version when you make backward compatible bug fixes.
Default value is 0.

VersionConfig.buildTimeStamp

buildTimeStamp = NaN

Specifies the timestamp that corresponds to the build date for the app.
Default value is NaN.
